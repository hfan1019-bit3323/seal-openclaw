# 市场新闻雷达

把新闻流变成投研输入，不把资讯堆砌成噪音。

## 目标

围绕 A 股投研场景，做三件事：
- 收集
- 去重
- 提炼出真正影响盘面的变量

## 当前环境约束

- 本地优先使用公开网页和 RSS 思路，不依赖必须联网鉴权的新闻 API。
- 暂时先做“人工触发可用”的 skill；持续监控和定时推送后续再接 cron。
- 没有稳定 RSS feed 时，可用栏目页/新闻列表页替代。

## 新闻分类框架

收到新闻后，优先归到以下类别：
1. 宏观
2. 政策监管
3. 商品 / 能源 / 地缘
4. 行业景气
5. 公司公告 / 公司新闻
6. 海外市场传导

## 判断新闻值不值得写进简报

只有满足以下任一条件，才进入摘要：
- 可能影响次日或本周盘面
- 可能改变某个板块的景气判断
- 可能触发资金风格切换
- 与用户关注方向高度相关
- 是官方表态、硬数据、业绩、订单、价格、政策级别变化

以下内容默认降权：
- 重复转载
- 无数据支撑的观点稿
- 标题党
- 与盘面无关的泛娱乐式资讯

## 通用执行流程

1. 明确时间范围：隔夜、今日、近三日、本周。
2. 收集候选资讯：
   - 官方源
   - 财经媒体
   - 行业媒体
   - 公司公告/新闻
3. 对需要精读的链接，优先用 `opencli-fetch` 抽正文；若不可用，再降级到 `web_fetch`。
4. 去重：同一事件只保留最原始或信息量最大的版本。
5. 分类：按宏观、政策、行业、公司、海外传导整理。
6. 评估市场影响：
   - 正向 / 负向 / 中性
   - 影响板块
   - 影响期限（盘中 / 1-3天 / 中期）
6. 输出高信息密度摘要。

## 默认输出格式

### 一、今日最重要的 3-5 条
每条都写：
- 事件
- 为什么重要
- 影响到什么板块 / 风格

### 二、分类整理
- 宏观
- 政策
- 商品 / 地缘
- 行业
- 公司

### 三、市场含义
- 哪些方向可能受益
- 哪些方向可能承压
- 哪些信息仍需验证

### 四、后续跟踪
- 明天 / 本周要继续看的数据或事件

## 晨报模式

如果用户要盘前简报，优先关注：
- 隔夜美股与中概
- 原油 / 黄金 / 美元指数 / 关键商品
- 国内政策和监管消息
- 龙头公司公告
- 可能影响开盘情绪的地缘或宏观事件

## 盘后模式

如果用户要盘后复盘素材，优先关注：
- 当日盘面热点对应的消息催化
- 龙虎榜和资金行为能否被消息解释
- 晚间公告对次日的潜在影响

## 周报模式

如果用户要周度总结，重点不是堆新闻，而是提炼：
- 这一周主线变化
- 催化是否持续
- 哪些新闻被市场真正定价
- 哪些是噪音

## 与其他 skill 的关系

- 找新闻源、搜专题时先用 `cn-market-search`
- 抽新闻正文时优先用 `opencli-fetch`
- 需要把新闻映射到板块和资金行为时，用 `a-share-analysis`
- 需要解析新闻附带 PDF/报告时，用 `report-parser`
