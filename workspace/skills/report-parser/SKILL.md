# 研报与文档解析

把 PDF、公告、研报、长文解析成投研可用的结构化摘要。

## 目标

在当前本地环境下，优先处理：
- 可直接读取的文本 PDF
- 用户粘贴的长文
- 网页文章

对扫描件 OCR、复杂表格、多栏版式、图片型 PDF，先做降级说明；后续接入更强 PDF/OCR 能力后再增强。

## 当前环境约束

- 当前没有稳定的专用 PDF 解析栈，不要承诺完美提取。
- 如果是本地 PDF 文件：优先尝试直接读取；读不出时明确说明可能是扫描件或版式复杂。
- 如后续接入 MinerU / OCR / 文档 API，可把这份 skill 作为上层输出规范继续使用。

## 输入类型与处理顺序

### 1. 用户直接给 PDF 文件
- 先尝试读取文本层。
- 如果文本质量差、乱码多、内容残缺：明确标记为“解析不完整”。
- 不完整时优先提取：标题、作者/机构、摘要、章节、关键表格附近文字。

### 2. 用户给网页 / 链接
- 优先使用 `opencli-fetch` 清洗网页正文；如 OpenCLI browser bridge 未就绪，再降级到 `web_fetch`。
- 如网页是研报摘要页，重点提取核心观点、预测、评级、关键数据。

### 3. 用户直接粘贴文本
- 直接进入结构化摘要，不必重复原文。

## 通用执行流程

1. 确认文档类型：研报、公告、新闻深度稿、财报摘录、会议纪要。
2. 提取基础信息：标题、来源、作者/机构、日期、覆盖对象。
3. 判断文档核心目的：
   - 行业逻辑
   - 公司推荐 / 跟踪
   - 业绩解读
   - 事件点评
   - 政策解读
4. 提取关键内容：
   - 核心观点
   - 关键数据与预测
   - 利好 / 利空驱动
   - 风险提示
5. 结合海豹定位，补一层“所以呢”：
   - 对哪些板块有影响
   - 对哪些类型公司有影响
   - 这属于短期催化还是中期逻辑
6. 输出结构化结论。

## 默认输出格式

### 一、文档概览
- 标题：
- 来源 / 机构：
- 日期：
- 类型：
- 主题：

### 二、核心结论
用 3-5 条写最重要的观点。

### 三、关键数据
列出文中的数字、同比/环比、预测、估值、价格、产能、订单等。

### 四、影响链条
按下面结构写：

```text
事件 / 观点
→ 影响行业 / 环节
→ 可能受益或承压的公司类型
→ 对市场交易的意义（短期催化 / 中期趋势 / 估值修复）
```

### 五、风险点
- 文档里明确写的风险
- 结合盘面应补充的风险

### 六、给用户的下一步关注
- 需要继续验证的数据
- 值得跟踪的板块 / 公司 / 公告

## 研报专用增强

如果是券商/机构研报，额外提取：
- 评级变化
- 盈利预测变化
- 目标价
- 与市场当前主线是否一致
- 是否属于“明牌观点”还是“边际增量信息”

## 公告专用增强

如果是上市公司公告，额外提取：
- 公告类型（业绩预告 / 合同 / 减持 / 定增 / 回购 / 诉讼 / 停复牌等）
- 对收入、利润、现金流、订单、产能的直接影响
- 是否会影响次日情绪或板块联动

## 解析失败时的处理

出现以下情况时要明确告知：
- 文件无法读取
- 只有图片没有文本层
- 表格/公式丢失严重
- 只拿到摘要页而非全文

此时仍尽量输出：
- 已成功提取的部分
- 缺失信息会影响什么判断
- 用户如能补截图/原文，可继续细化

## 与其他 skill 的关系

- 需要从网页找报告时，先用 `cn-market-search`
- 需要抽取网页正文时，优先用 `opencli-fetch`
- 需要把报告结论映射到 A 股盘面时，再交给 `a-share-analysis`
