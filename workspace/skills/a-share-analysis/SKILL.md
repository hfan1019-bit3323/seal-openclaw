# A股市场分析

按北京时间分析 A 股，先给结论，再给数据和逻辑。

## 核心原则

- 统一使用北京时间（UTC+8）。
- 以 A 股自身节奏为主，不把美股映射当成默认主因。
- 优先引用原始或近原始数据源，不用搜索摘要替代数据。
- 数据不完整时明确标注“已确认”和“待确认”，不要补造。
- 不直接给买卖指令；输出判断、证据、风险点、后续关注项。

### 当前环境下的数据策略

**首选：Market Data Plane（缓存 ~6ms，结构化数据）**

板块资金流、龙虎榜、指数行情、涨停池、北向资金这 5 类数据，**优先走 `market-fetcher` skill**，不要第一步就上 agent-browser：

1. 先调 `market.read_latest(task_id)` 读缓存
2. 未命中则调 `market.fetch_now(task_id, akshare_fn, args)` 同步抓取
3. 用户反复问同一类数据且清单里没有 → 调 `market.schedule()` 后告知用户

已支持的 task_id：`sector-flows` / `dragon-tiger` / `index-daily` / `limit-up` / `north-flow`

Market Data Plane 不可用（连续 5xx 或 fetch 超时）时，降级到下方的 agent-browser 路径。

**降级/补充：web_search + agent-browser + opencli-fetch + web_fetch**

Market Data Plane 不覆盖的数据（个股资金、新闻、公告、同花顺等），按以下顺序：
1. 东方财富
2. 新浪财经
3. 同花顺
4. 证券时报/数据宝
5. 英为财情

如需更多背景，再读：
- `references/data-sources.md`：数据源与搜索模板
- `references/quant-detection.md`：量化/程序化资金识别
- `references/sector-rotation.md`：板块轮动与过热/成主线信号

如需抽取网页正文，按以下优先级：
1. **JS 渲染页面**（龙虎榜、板块资金流、个股资金流向等需要 JavaScript 才能加载数据的页面）：
   - 优先用 `agent-browser`（路径：`/Volumes/Data2/ai/.tooling/npm-global/bin/agent-browser`）
   - 示例：`agent-browser fetch "https://data.eastmoney.com/lhb/default.html"`
   - 可配合 `--wait-for` 等待数据渲染，`--screenshot` 用于调试
2. **静态页面**（新闻、公告、研报等直出 HTML）：
   - 优先用 `opencli-fetch`
   - OpenCLI 不可用时降级到 `web_fetch`

**哪些页面必须走 agent-browser（禁止直接 fetch）：**
- 东方财富龙虎榜：`data.eastmoney.com/lhb/`
- 东方财富板块资金流：`data.eastmoney.com/bkzj/`
- 东方财富个股资金流：`data.eastmoney.com/zjlx/`
- 同花顺资金流：`fund.10jqka.com.cn/`
- 任何 URL 返回空内容或 JS 框架占位符时，立即切换到 `agent-browser`

## 通用执行流程

1. 确认问题属于盘中、收盘后、预判次日，还是个股/板块专项分析。
2. 确认当前或目标交易日，并显式写出时间锚点，如“截至 4 月 13 日收盘”。
3. 收集最少必要数据：指数、成交额、涨跌家数、板块资金、龙虎榜/席位、关键消息催化。
4. 先做归因，再做分类：
   - 产业逻辑
   - 事件催化
   - 资金性质（机构/北向/量化/游资/散户）
   - 行情属性（脉冲/持续/分化）
5. 输出结论时始终包含：
   - 当前判断
   - 证据
   - 风险或反证
   - 用户下一步该关注什么

## 任务A：盘中/收盘资金流向快照

用于“看看今天盘面”“今天资金流向怎么样”“收盘分析”这类请求。

### 执行

1. 判断是盘中还是收盘后。
2. 获取指数表现、成交额、涨跌家数、涨停/连板/封板率。
3. 获取板块净流入前 5 和净流出前 5（用 `agent-browser` 抓取东方财富板块资金流页面）。
4. 获取龙虎榜关键信号（用 `agent-browser` 抓取，JS 渲染页面）：机构净买入/净卖出、北向动向、典型营业部席位。
5. 如尾盘异常放量或多板块同步异动，再读取 `references/quant-detection.md` 判断量化调仓痕迹。
6. 如板块轮动明显，再读取 `references/sector-rotation.md` 做交叉验证。

### 输出格式

- 大盘概况：不超过 3 行
- 资金流入方向表：板块 | 代表个股 | 资金性质 | 结论
- 资金流出方向表：板块 | 代表个股 | 资金性质 | 结论
- 量化资金判断：1 段
- 后续关注：2-4 条

## 任务B：识别正在被悄悄建仓的板块

用于“哪个板块在建仓”“谁在悄悄买”“接下来可能轮到谁”。

### 建仓判定标准

仅当以下三项同时满足时，才定义为“建仓中”：
1. 连续多日温和净流入，而不是单日爆量脉冲。
2. 成交量温和放大，而不是极端换手。
3. 价格仍处于底部或主升前段，而非已大幅明牌。

### 执行

1. 扫描近 1 周到 1 个月的持续净流入候选板块。
2. 结合催化、产业链联动、龙虎榜席位判断资金质量。
3. 用任务C的因果链框架解释“为什么是它”。
4. 用任务D的框架判断是建仓延续还是短炒脉冲。
5. 对候选方向按强度排序，并区分：
   - 正在建仓
   - 值得观察
   - 已经明牌/过热

### 输出格式

- Top 3 建仓方向：方向 | 强度 | 驱动因果链 | 当前阶段 | 关注个股
- 观察名单
- 过热方向与风险提示

## 任务C：因果链分析

用于“为什么涨/跌”“是外盘带的还是自身逻辑”“根因是什么”。

### 分析框架

```text
根因（产业周期 / 政策 / 事件）
  → 传导机制（涨价 / 业绩 / 估值重定价）
  → 资金行为（机构 / 北向 / 量化 / 游资 / 散户）
  → 股价表现（脉冲 / 持续 / 分化）
```

### 必答问题

1. 是否受美股影响；如果是，影响权重有多大。
2. 是否受油价/中东等外部事件影响；如果是，属于催化器还是根因。
3. 因果链强度几星，持续性如何。

## 任务D：脉冲 vs 持续判断

用于“这波能不能持续”“是游资脉冲还是机构趋势”。

### 主判断维度

| 维度 | 脉冲特征 | 持续特征 |
|------|---------|---------|
| 谁在买 | 游资主导，机构偏卖出 | 机构净买入，机构+北向共振 |
| 换手率 | 常见于极端高换手 | 常见于 10%-20% 的正常放量 |
| 催化性质 | 一次性消息或情绪刺激 | 产业周期、涨价、业绩、政策持续催化 |

### 附加验证

- 次日是否续强或快速开板
- 产业链是否联动，而非单股表演
- 大盘走弱时能否逆势
- 龙虎榜席位是否显示机构/北向持续参与

只看涨跌幅不下结论；如能查到龙虎榜，必须纳入。

### 持续性判断模块

当用户追问“为什么判断会持续”“能持续多久”“这是不是一日游”时，统一按下面 5 个维度输出，不要只凭感觉回答。

#### 维度 1：连续性
看资金和价格是否连续，不看单日异动。
- 今日是否净流入
- 次日/近 3 日是否继续净流入
- 是否连续出现在板块资金前列

判断：
- 只有单日流入 → 只能算强化迹象
- 连续 2-3 天流入 → 才能开始讨论短趋势
- 连续 5 天以上流入且递增 → 才能视为中短期主线候选

#### 维度 2：扩散性
看是不是从龙头扩散到板块和产业链。
- 只有 1-2 只龙头上涨，其他不跟 → 偏脉冲
- 细分板块同步走强 → 持续性更高
- 上中下游或相关配套一起动 → 说明不是单点炒作

#### 维度 3：资金质量
看是谁在买，而不是只看金额。
- 机构净买入、机构席位增加 → 趋势属性更强
- 北向或机构+北向共振 → 持续性加分
- 主要由游资推动、机构反手卖出 → 更偏短脉冲

#### 维度 4：风格兼容性
看这个方向是否与当前市场主风格一致。
- 若当前市场主风格是科技成长，而该板块也属于科技/高端制造/景气成长 → 更容易持续
- 若当前市场在防守，而该板块属于高弹性进攻题材 → 持续性会受限
- 若板块能与当日最强主线共振，而不是与其争夺资金 → 更容易延续

#### 维度 5：抗压性
看板块在分歧日和弱市中的表现。
- 大盘震荡或回落时仍能逆势走强 → 持续性高
- 一旦市场分歧就大幅回落 → 更像情绪脉冲
- 龙头不掉队、板块不明显缩圈 → 持续性加分

### 持续性分级标准

按以下标准给结论，不要模糊表述：

- **脉冲**：单日资金流入，缺乏扩散，机构参与弱，次日承接存疑
- **强化迹象**：当日板块级流入明确，细分共振开始出现，但连续性未验证
- **短趋势**：连续 2-5 个交易日有资金流入，板块扩散，龙头稳，机构有参与
- **阶段主线候选**：连续多日流入递增，产业链联动，机构/北向共振，弱市也能抗跌

### 回答模板

默认按下面结构回答：

1. **当前级别**：脉冲 / 强化迹象 / 短趋势 / 阶段主线候选
2. **判断原因**：从连续性、扩散性、资金质量、风格兼容性、抗压性五个维度各写 1 句
3. **可持续多久的判断**：
   - 仅日内
   - 2-5 个交易日
   - 1-2 周
   - 仍需观察，无法下结论
4. **失效条件**：例如次日资金转负、龙头掉队、龙虎榜机构转卖、板块缩圈

## 任务E：交易日预判

用于“明天怎么看”“周一预判”“下周方向”。

### 执行

1. 锁定最近一个交易日的收盘数据。
2. 检查隔夜外盘、油价、黄金、美元指数及重要政策/地缘消息。
3. 判断这些变量对 A 股的传导方向和强弱。
4. 明确写出 3-4 个情景及概率，不做单路径叙事。
5. 输出板块级判断，而不是只谈指数。

### 预判时重点关注

- 成交额是否站上多空分水岭
- 周五尾盘结构是否对周一有延续性
- 月末/季末机构调仓压力
- 财报披露期的业绩雷风险
- 前一交易日的主线是否出现龙头掉队、板块缩圈、机构分歧

不要把固定点位写成永恒规则；如引用关键支撑/压力位，注明这是特定阶段观察值。

## 结果表达规范

默认按下面结构回答：

1. **结论**：一句话先说判断。
2. **关键证据**：3-5 条，带数据或来源类型。
3. **资金性质判断**：机构 / 北向 / 量化 / 游资谁主导。
4. **风险与反证**：什么情况下当前判断失效。
5. **下一步关注**：用户接下来该盯什么数据或事件。

## 数据不足时的处理

如果拿不到当天关键数据：
- 明确说缺了什么数据。
- 说明这会影响哪部分判断。
- 尽量给出基于已知信息的低置信度结论。
- 必要时请求用户补充截图、代码、板块名或个股名。
