# 中文市场搜索编排

用这个 skill 解决“怎么搜”和“搜什么”两个问题，不负责虚构数据。

## 目标

把用户的模糊需求，转成适合 `web_search` / `web_fetch` 执行的中文财经搜索流程。

## 当前环境约束

- 优先使用内置 `web_search` 与 `web_fetch`。
- 当前宿主浏览器能力可能不可用；不要默认依赖浏览器抓取。
- 遇到反爬、JS 渲染或结果缺失时，切换来源、缩短 query、拆分任务，不要强行编造结论。
- 不做激进绕过：不默认伪装浏览器、不主动获取 cookie、不做灰色爬虫策略。

## 数据源优先级

按任务类型选择来源，不要一把梭。

### A股盘面 / 资金流 / 龙虎榜
1. 东方财富
2. 新浪财经
3. 同花顺
4. 证券时报 / 数据宝
5. 21世纪经济报道等财经媒体作补充

### 政策 / 公告 / 官方口径
1. 证监会 / 交易所 / 国务院 / 发改委 / 工信部官网
2. 上证报 / 中证报 / 证券时报
3. 东方财富 / 新浪财经二次整理

### 商品 / 海外传导
1. 英为财情
2. 财经媒体快讯页
3. 官方或交易所页面

## 通用执行流程

1. 明确任务类型：
   - 收盘复盘
   - 次日预判
   - 板块资金流
   - 龙虎榜/机构席位
   - 公司公告
   - 政策催化
   - 商品与外盘联动
2. 补齐时间锚点：默认写明日期，必要时加“收盘”“午盘”“盘后”。
3. 先搜概览，再搜专项，不要一上来就搜很细。
4. 至少找 2 个来源交叉确认关键结论。
5. 输出时区分：
   - 已确认
   - 待确认
   - 无法确认

## Query 模板

### 收盘 / 午盘概览
- `"{日期}" A股 收盘 沪指 深成指 创业板 成交额 板块`
- `"{日期}" A股 午评 沪指 深成指 创业板 成交额`

### 板块资金流
- `"{日期}" 主力资金流向 板块 净流入 净流出 A股`
- `"{日期}" 行业资金 净流入 东方财富`

### 龙虎榜 / 机构 / 北向
- `"{日期}" 龙虎榜 机构 净买入 北向 A股`
- `"{个股名}" {日期} 龙虎榜 机构 营业部`

### 涨停 / 连板 / 封板率
- `"{日期}" 涨停分析 连板 封板率 A股`

### 政策 / 事件催化
- `"{日期}" {主题词} 政策 A股`
- `"{日期}" {主题词} 工信部 发改委 证监会`

### 商品 / 外盘
- `"{日期}" 美股 收盘 纳指 标普 道指 原油 黄金 美元指数`
- `"{日期}" 碳酸锂 价格 原油 价格`

## 拆分策略

如果一个 query 太大，拆成三层：
1. 市场概览
2. 板块 / 个股专项
3. 佐证来源

例子：
- 先搜：`"4月13日" A股 收盘`
- 再搜：`"4月13日" 主力资金流向 板块`
- 再搜：`"4月13日" 龙虎榜 机构 北向`

## 降级策略

### `web_search` 失败或被 challenge
- 缩短 query
- 去掉多余限定词
- 直接 `web_fetch` 关键站点栏目页
- 改搜二级来源验证结论

### 页面是 JS 渲染 / 文本抓不下来
- 改抓同主题的新闻稿、午评、收评页
- 改抓网站列表页或专题页
- 明确告知“当天明细未完整获取”

### 结果互相矛盾
- 优先保留原始/近原始来源
- 用“主流媒体整理”做旁证，不直接覆盖原数据

## 输出要求

默认输出四段：
1. 搜索结论
2. 已确认的数据点
3. 交叉验证来源类型
4. 仍缺的数据与影响

## 不要做的事

- 不把搜索摘要当最终事实
- 不只凭单一媒体标题下结论
- 不为凑完整性编数据
- 不把搜索 skill 当成行情数据库

## 参考方向
